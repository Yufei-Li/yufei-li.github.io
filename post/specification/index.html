catogories of specification

requirements specification

  • specify what piece of software is supposed to do before writing software
  • a collabration of software producer and software user

formal specification

  • mathmatical expression of what software is going to do and how will it work

interface specification

  • is related to api , how to use a library

protocal specification

  • make sure the work between two programmers can interpretted correctly

semantics and syntax

  • to achieve interoperability, we need to specify the semantics and syntax

semantics

  • explain what do the message mean

  • Example:

    • when do the program send the message? / how often to send message?
    • what does the message mean?
    • what should recipient do on receipt?

  • semantics for pac-man game:

  • network protocol:

    • UDP: fast, but sometimes lossy

      • when using UDP, need to think how to handle with some messages that be lost, since some information is important. e.g. if the pacman dies the other side, need to know it died and stop animating

    • TCP: reliable, sometimes slow

      • when using TCP, need to think how to deal with late information. The pacman might jump backward

    • for UDP & TCP :https://www.cnblogs.com/fundebug/p/differences-of-tcp-and-udp.html

syntax

  • explain how is the information encoded

  • use a textual encoding

  • advantages:

    • the encoding is readable
    • great amount of flexibility

  • disadvantages:

    • Inefficient for computer, use more internal capacity

  • e.g. url is a textual encoding to locate the information on the Internet. the basic specification of url is the character must be a ascii, if you wanna add a unicode, you need to maps it into ascii, but after mapping, it is not really readable (Punycode)

  • use a binary encoding
