Urllib笔记

urllib包含四个模块：

  1. request，是最基本的http请求模块，可以用来模拟发送请求

  2. error，异常处理模块，如果出现请求错误，可以进行操作保证程序不会意外终止

  3. parse，工具模块，提供了许多url处理方法

  4. robotparser，识别robots.txt文件，判断网站的可爬性，使用较少

request模块

urlopen

  • get请求：通过网址传递参数直接打开页面，请求页面

    1
2
3

        import urllib.request
    response = urllib.request.urlopen("http://www.baidu.com")
    print(response.read().decode('utf-8'))

    urllib.request.urlopen可以将网页的源代码装进response里，可以使用.read()来阅读，使用.decode(‘utf-8’)来解码，使中文正常显示

    1
2
3
4
5

      response = urllib.request.urlopen("http://www.baidu.com")
  print(type(response))  #<class 'http.client.HTTPResponse'>
  print(response.status)   #返回状态码
  print(response.getheaders())   #返回header里具体内容
  print(response.getheader("Server"))   #返回Server属性对应的内容

    response是一个HTTPResponse类型的对象，主要的方法有read(), readinto(), getheader(name), getheaders(), fileno()等方法和msg, version, status, reason, debuglevel, closed等属性

  • post请求（data参数）：传递byte形式的data，内容不在网址上体现

    1
2
3
4

        import urllib.parse    #解析键值对，用utf-8的方式解析，再统一变成二进制
    data = bytes(urllib.parse.urlencode({"hello": 'world'}), encoding="utf-8") #转化为bytes来封装数据
    response = urllib.request.urlopen("http://httpbin.org/post", data=data)
    print(response.read().decode("utf-8"))

    使用urllib模拟了在浏览器里真实发出的请求，parse的urlencode将字典转化为字符串

  • timeout参数：可以设置超时时间，单位是秒

    print(response.read().decode("utf-8"))
except urllib.error.URLError as e:
    print('time out!')

    如果超出一定时间没有打开网页，就报错并且进行下一网页的爬取

Request

使用urlopen可以实现最基本的请求，但是我们可以使用Request类来构建一个更完整的请求。Request的参数如下：

class urllib.request.Request(url, data=None, headers={}, origin_req_host=None, unverifiable=False, method=None)

第一个参数是url，是必传参数

data参数必须要传bytes类型，如果是一个字典，可以先用urllib.parse模块的urlencode()转换成字符串

headers的参数必须是一个字典，最常用的方法是修改User-Agent来伪装浏览器，在开发者模式中可以显示headers的信息，发出的请求越贴近这些信息，就会更像是浏览器发出的请求而不是爬虫

origin_req_host是指请求方的host名称或者IP地址

unverifiable参数指的是这个请求是否是无法验证的，默认是False，意思是说用户没有足够权限来选择接收这个请求的结果，例如我们请求一个 HTML 文档中的图片，但是我们没有自动抓取图像的权限，这时 unverifiable 的值就是 True

method参数是一个字符串，它用来指示请求使用的方法，比如GET，POST，PUT等等

1
2
3
4
5
6
7

   url = "https://www.douban.com"
   headers = {
   "User-Agent": "Mozilla/5.0 (Windows NT 6.1; ) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/81.0.4044.129 Safari/537.36"
   } #headers将信息以键值对的形式封装
   req = urllib.request.Request(url=url, headers=headers)   #封装请求，还可以添加data= ， method="POST"等等
   response = urllib.request.urlopen(req)   #req是一个请求，需要再对它使用urlopen来查看response
   print(response.read().decode("utf-8"))   #解码查看

在headers里的信息越接近上面开发者模式中的信息，越不容易被识别出来。在这里对于豆瓣的范围，只伪装了user-agent

error模块

Urllib 的 error 模块定义了由 request 模块产生的异常。如果出现了问题，request 模块便会抛出 error 模块中定义的异常。 处理异常的类主要有两个 URLError 和 HTTPError ，URLError 是 HTTPError 的父辈

  • HTTPError有三个属性

    1. code，返回 HTTP Status Code，即状态码，比如 404 网页不存在，500 服务器内部错误等等

    2. reason，返回错误的原因

    3. headers，返回 Request Headers

1
2
3
4
5
6
7
8

 try:
 response = request.urlopen('http://baidu.com')
 except error.HTTPError as e:
 print(e.reason, e.code, e.headers, sep='\n')
 except error.URLError as e:
 print(e.reason)
 else:
 print('Request Successfully')

这样我们就可以做到先捕获 HTTPError，获取它的错误状态码、原因、Headers 等详细信息。如果非 HTTPError，再捕获 URLError 异常，输出错误原因。最后用 else 来处理正常的逻辑，这是一个较好的异常处理写法。

parse模块

解析url

  • urlparse()：将URL解析成一个元组，元组中包含六个元素，协议(scheme)，域名(netloc)，路径(path)，路径参数(params)，查询参数(query)，片段(fragment)

  • urlsplit()：也可以对url进行拆分，元组中包含五个元素，urlsplit()不会吧路径参数(params)从路径(path)中分离出来

  • urldefrag():解析成一个元组，只包含url和fragment

组建url

  • urlunparse()接收一个列表的参数，而且列表的长度必须在六个以上

    1
2

      url_compos = ('http', 'user:pwd@domain:80', '/path1;params1/path2', 'params2', 'query=queryarg', 'fragment')
  print(urllib.parse.urlunparseurlunparse(url_compos))

    结果为http://user:pwd@domain:80/path1;params1/path2;params2?query=queryarg#fragment

  • urljoin():将两个字符串拼接成url

    1
2
3

      # 连接两个参数的url, 将第二个参数中缺的部分用第一个参数的补齐,如果第二个有完整的路径，则以第二个为主
  print(urllib.parse.urljoin('https://movie.douban.com/', 'index'))
  print(urllib.parse.urljoin('https://movie.douban.com/', 'https://accounts.douban.com/login'))

    结果为

    1
2

      https://movie.douban.com/index
  https://accounts.douban.com/login
